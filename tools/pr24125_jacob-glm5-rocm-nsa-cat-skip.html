[AMD] perf · rocm-nsa · GLM-5

Skip the redundant CatArrayBatchedCopy
in NSA TileLang decode.

fused_qk_rope_cat_and_cache_mla already returns a contiguous q_cat; the downstream concat_mla_absorb_q_general rebuilds a byte-identical copy. On ROCm that copy fires once per layer per decode step as CatArrayBatchedCopy — pure waste.
01
The Redundancy · memory view

One block exists. The kernel allocates another.

Pre-patch · ROCm decode 576 dims × 2 buffers
Post-patch · HIP only 576 dims × 1 buffer

Why does the redundant cat exist at all? The attn_mqa contract takes (q_nope, q_rope) but every NSA decode impl downstream takes q_all. On CUDA the gap is closed by a fused concat_mla_absorb_q kernel; on ROCm concat_mla_absorb_q_general falls through to torch.cat → ATen CatArrayBatchedCopy. The PR teaches both ends of the contract that, in HIP decode, the second buffer is unnecessary.

03
Reviewer Invariants · what stays the same

Six guardrails. All hold.

Zero-copy view

q.contiguous().view(...) on a fresh q_cat: shape (M,H,576) already matches target. Metadata-only.

Bit-exact q_all

Pre-patch torch.cat just copies q_cat[..,:512] + q_cat[..,512:] back. Skipping reads the same bytes.

llama_4_scaling safe

q_cat[..,:512] *= s aliases the slice view; identical to pre-patch q_nope_fused *= s.

CUDA / MUSA unchanged

Gate q_all is None or not _is_hip. On non-HIP _is_hip == False → always re-cats. Pre-patch verbatim.

Prefill unchanged

forward_extend asserts q_rope is not None; new caller branch keeps the split form for non-decode modes.

Split caller still works

q_all = None sentinel signals "still need to cat". All existing callers (CUDA + MUSA) take this path.
